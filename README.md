# DecodeLabs-DataAnalytics-Project3-
SQL sales data analysis project using MySQL Workbench to generate business insights from an e-commerce dataset.
🛒 Project 3: Sales Data Analysis Using SQL (MySQL)

📌 Project Overview

This project analyzes an e-commerce sales dataset using SQL in MySQL Workbench. The objective was to explore sales performance, customer purchasing behavior, payment methods, product performance, and overall business trends by writing SQL queries to answer key business questions.

🎯 Objectives

* Analyze total sales revenue and order volume.
* Evaluate customer purchasing patterns.
* Identify the most popular payment methods.
* Analyze order status and fulfillment performance.
* Determine the top-selling products.
* Examine referral sources and customer acquisition channels.
* Analyze monthly sales trends.
* Perform data quality checks to ensure reliable analysis.

📂 Dataset Information

The dataset contains 1,200 sales records with the following columns:

* OrderID
* Date
* CustomerID
* Product
* Quantity
* UnitPrice
* ShippingAddress
* PaymentMethod
* OrderStatus
* TrackingNumber
* ItemsInCart
* CouponCode
* ReferralSource
* TotalPrice

🛠️ Tools Used

* MySQL Workbench
* SQL
* Git & GitHub

💡 SQL Concepts Applied

* SELECT
* WHERE
* GROUP BY
* ORDER BY
* COUNT()
* SUM()
* AVG()
* DISTINCT
* LIMIT
* Aggregate Functions
* Data Quality Validation

📊 Business Questions Answered

* What is the total sales revenue?
* How many orders were placed?
* What is the average order value?
* Which payment methods are used most frequently?
* Which order statuses occur most often?
* Which products generate the highest sales revenue?
* Which products sell the highest quantities?
* Who are the top customers by spending?
* Which referral sources generate the most revenue?
* How do sales perform over time?
* Are there missing values, duplicate records, or invalid prices?

📈 Key Insights

* Calculated total sales revenue and average order value.
* Identified the most frequently used payment methods.
* Compared revenue across different payment methods.
* Evaluated order status distribution.
* Ranked products by quantity sold and revenue generated.
* Identified high-value customers.
* Measured the performance of referral sources.
* Analyzed monthly sales trends.
* Verified data quality by checking for missing values, duplicate OrderIDs, and invalid prices.

✅ Conclusion

This project demonstrates practical SQL skills for data analysis using MySQL Workbench. It highlights the ability to clean, explore, aggregate, and analyze business data to generate actionable insights that can support business decision-making.
